Status

Nx86 has completed 33 development phases (Phase 0 through Phase 32). The compiler pipeline, runtime, memory system, and GUI shell are all functional. The project cannot run real Switch games yet.

Now

  • Synthetic ARM64 programs travel the full compiler pipeline: decode, lift to NxIR, optimize, verify, lower to x86_64, package as .nxo.
  • NxIR exists as a typed SSA intermediate representation with a verifier and dead-flag elimination.
  • Multi-block native execution through a dispatcher with guest PC lookup and block exit handling.
  • Emergency JIT compiles missing blocks on the fly and adds them to the cache.
  • Block chaining with guarded native exits and W^X patching (feature-gated).
  • Fastmem: direct guest memory access through a 64 GiB mmap arena with checked fallback.
  • Slowmem: fallback load/store helpers with counters, fault reporting, and coverage penalties.
  • Memory mirroring with release/debug mode support.
  • Guard and deopt metadata for future speculative optimization.
  • Cache manager with .nxo integrity checking, manifest, and atomic writes.
  • Runtime profiling: versioned JSONL profiles with typed events, crash-tail recovery, and file locking.
  • Profile-guided rebuild promotes JIT-discovered blocks to AOT objects.
  • CFG recovery via recursive-descent traversal.
  • Inspector with disassembly, CFG, NxIR dump, and native mapping views.
  • GUI shell with Library, Compile, Tests, Settings, and Inspector screens (egui, Linux).
  • 215 differential tests — interpreter, NxIR evaluator, and native execution must all agree.

Not yet

  • No public downloads.
  • No game import or title boot.
  • No firmware, keys, or bundled system files.
  • No homebrew execution yet.
  • No floating-point or SIMD/NEON support.
  • No conditional branch lowering in native multi-block functions.
  • No atomics or barrier semantics.
  • No guest threading.
  • No Vulkan rendering or shader translation.
  • No Windows or macOS native execution.

Next

  • Phase 33: Self-Modifying Code Support — executable page write detection, object and chain invalidation.
  • Phase 34: Atomics v0 — exclusive monitor model, LDXR/STXR subset, acquire/release.
  • Phase 35: Barrier Semantics v0 — DMB/DSB/ISB subset.
  • Phase 36: Guest Threading v0 — thread state isolation, scheduler skeleton.
